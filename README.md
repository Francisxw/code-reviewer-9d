# Code Reviewer 9D

> 一个基于 FAANG Staff+ 工程师标准的代码审查工具，使用 9 维度检查清单进行严格的代码质量评估。

## 📋 简介

Code Reviewer 9D 是一个专业的代码审查技能，通过系统化的 9 个维度对代码进行全面评估。它可以帮助你发现代码中的潜在问题、安全漏洞、性能瓶颈，并提供具体的改进建议和代码重构方案。

## ✨ 核心特性

- **9 维度全面审查**：从健壮性、边界情况、命名规范等 9 个角度深入分析代码
- **FAANG 级别标准**：采用一线科技公司的高级工程师审查标准
- **详细的问题报告**：提供具体的问题定位、风险评估和修复方案
- **整体评分系统**：基于多维度评估给出 1-10 分的整体代码质量评分
- **实用的重构建议**：不仅指出问题，还提供优化的代码示例

## 🎯 9 大审查维度

| 维度 | 说明 |
|------|------|
| 1. **健壮性** (Robustness) | 安全漏洞、类型安全、算法效率、资源泄漏 |
| 2. **边界情况** (Edge Cases) | 错误处理、输入验证、加载状态、取消操作 |
| 3. **命名规范** (Naming) | 清晰度、一致性、描述性 |
| 4. **注释质量** (Comments) | 适当使用、清晰度、避免过度注释 |
| 5. **代码复用** (Reuse) | 使用现有库、提取共享逻辑、消除重复 |
| 6. **代码组织** (Colocation) | 相关文件存储、逻辑模块边界 |
| 7. **职责分解** (Decomposition) | 单一职责、复杂度控制、关注点分离 |
| 8. **一致性** (Consistency) | 遵循项目模式、编码标准 |
| 9. **测试覆盖** (Tests) | 清晰性、原子性、速度、覆盖率 |

## 🚀 使用方法

### 触发关键词

当你使用以下任一关键词时，将自动触发代码审查：

- `review`
- `code review`
- `audit`
- `check this code`
- `feedback on PR`
- `is this code good`
- `what's wrong with`
- 或任何请求评估代码质量的表达

### 使用示例

```
请帮我 review 这段代码：

function getData(id) {
  const data = db.query(`SELECT * FROM users WHERE id = ${id}`);
  return data;
}
```

### 审查流程

1. **上下文收集**：理解代码语言、目标、范围
2. **9 维度系统审查**：逐维度检查并提供具体反馈
3. **结果综合**：生成详细报告和改进建议

## 📊 评分标准

| 分数 | 含义 |
|------|------|
| **9-10** | 生产就绪，典范代码。仅可能有微小的风格问题 |
| **7-8** | 稳健的代码，有轻微问题。可在注明修复后发布 |
| **5-6** | 功能正常但有明显缺陷。发布前需要改进 |
| **3-4** | 重大问题。需要主要修复才能用于生产 |
| **1-2** | 关键问题。安全漏洞、崩溃或基础设计缺陷 |

## 🔧 技术栈

- 语言：多语言支持（JavaScript/TypeScript、Python、Java、Go 等）
- 应用场景：代码审查、PR 审核、质量审计、技术债务分析

## 📝 输出报告格式

每次代码审查都会生成结构化的报告，包括：

- **维度评分表**：9 个维度的评估结果
- **整体评分**：1-10 分的综合评分
- **前 3 个关键问题**：按优先级排序的严重问题
- **详细发现**：每个问题的具体说明和修复建议
- **重构建议**：针对需要重写的部分提供完整改进方案

## 💡 最佳实践

1. **提供完整上下文**：尽可能提供完整的文件或 PR 链接
2. **说明代码目标**：简要描述代码要实现的功能
3. **包含相关测试**：如果有测试文件，一并提供审查
4. **开放心态**：接受批评性反馈，这是提升代码质量的机会

## 🎓 设计理念

- **直接坦率**：明确指出问题，不绕弯子
- **具体明确**：引用具体的行号、函数名和模式
- **区分优先级**：区分"必须修复"（bug、安全）和"应该修复"（风格、可读性）
- **建设性**：不仅指出问题，还提供解决方案
